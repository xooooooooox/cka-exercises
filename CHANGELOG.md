# Changelog

All notable changes to the CKA practice repository — **webapp, exercise corpus, AND documentation** — are recorded here. New entries land under `## [Unreleased]` until a tagged release.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/). Categories used:

- **Added** — new features, new exercises, new docs sections
- **Changed** — behavior or wording changes that aren't bug fixes
- **Fixed** — bug fixes
- **Removed** — deleted features, deprecated docs

Each entry references the commit hash in parens for traceability to git history. Root-cause notes / migration details live in commit messages; the changelog stays single-line and user-visible.

## [v0.1.0] - 2026-06-25

### Added
- Documentation sync discipline codified in CLAUDE.md (new `## Documentation sync discipline` section). The rule: every code change must touch any reference docs that describe the changed behaviour, file structure, or user-facing flow — in the same commit. Triggers enumerated (Repository Layout / CI workflows / SPA features / localStorage keys / Build pipeline / Release mechanism / doc-structure changes). Changelog discipline remains separate — it's a change log, not a reference. (this commit)
- SPA version chip distinguishes **release** vs **dev** builds. `scripts/build-exercises.mjs` now stamps `channel` / `commitsAhead` / `gitSha` into `docs/version.json` via `git describe --tags --abbrev=0` + `git rev-list --count`. The header chip renders `vX.Y.Z` (default colour) when HEAD sits on a release tag with matching `package.json.version`, or `vX.Y.Z+dev.N` (subtle orange) for any deploy that isn't on a release tag — N = commits ahead of the last release. ✨ Refresh banner upgraded to show full composed labels so users see whether they're crossing a release boundary or just bumping `+dev.N`. `build-and-deploy-docs.yml` checkout gains `fetch-depth: 0` so the git fingerprint commands can resolve historical tags. (this commit)
- App-Store-style release pipeline. `scripts/release.mjs` reads `CHANGELOG.md`'s `[Unreleased]` block, infers a semver bump from its section composition (Removed/BREAKING → minor in v0.x or major in v1+; Added/Changed → minor; Fixed-only → patch), rewrites the changelog to rename `[Unreleased]` → `[vX.Y.Z] - YYYY-MM-DD` while prepending a fresh empty `[Unreleased]`, bumps `package.json`, commits, tags, pushes, and files a GitHub Release with the freshly-named version block as release notes. Manual `--bump=major|minor|patch|auto` override + `--dry-run` flag for previewing. `.github/workflows/release.yml` exposes the same controls via `workflow_dispatch` so maintainers cut releases from the Actions UI. (this commit)
- Always-visible `vX.Y.Z` version chip in the header right — monospace pill that shows the bundled SPA's semver tag (read from `docs/version.json`, baked in at build time by `scripts/build-exercises.mjs` reading `package.json.version`). Click jumps to Help → Changelog. Refresh banner now shows `v0.1.0 → v0.1.1` style version delta when a newer deploy is available instead of just "✨ New content available". (this commit)
- Service worker brings offline support to the iOS PWA. `docs/sw.js` (templated; `__BUILD__` placeholder is replaced at build time by `scripts/build-sw.mjs` reading `docs/version.json.generatedAt`, written to gitignored `docs/sw.gen.js`) precaches the SPA shell (`index.html` / `app.js` / `style.css` / `sync.js` / `llm.js` / `manifest.webmanifest` / icons) on install and serves it cache-first afterwards; JSON data files (exercises.json, tools-*.json, nodes-*.json) go through a stale-while-revalidate path so users see content instantly and the latest copy lands for the next visit; navigation requests fall back to the cached `index.html` when offline (closes the "iPhone home-screen icon → white screen" gap when network is dead on cold start). New deploy bumps the cache key → `activate` deletes old caches → next visit calls `controllerchange` which triggers one soft reload. Registration is skipped on localhost / 127.0.0.1 so `npm run serve` development isn't pinned to stale cache. (this commit)
- One-time install-to-home-screen hint banner shown 8 s after first load to users who aren't already in PWA standalone mode. iOS gets "Safari Share → Add to Home Screen"; everyone else gets "browser menu → Add to Home screen". Dismissal persists via `cka:install:dismissed`; suppressed permanently in `display-mode: standalone`. (this commit)
- Sidebar custom tooltip now responds to touch: long-press (600 ms) on a clipped sidebar row pops the full title on iPhone / iPad — previously touch devices saw nothing because the tooltip was hover-only. (this commit)
- Weekly CI watcher for the upstream CNCF curriculum. `.github/workflows/curriculum-watch.yml` cron-runs `scripts/check-curriculum.mjs` every Monday 13:00 UTC — it lists `CKA_Curriculum_v*.pdf` files in [cncf/curriculum](https://github.com/cncf/curriculum) via the GitHub Contents API, MD5s the known versions (v1.32-v1.35, baseline `d28372581378e1ff3aa49670a9c7781f` per EXAM_GUIDE), and files a labelled `curriculum-watch` GitHub issue when either (a) a new version PDF appears (e.g. `CKA_Curriculum_v1.36.pdf`) or (b) one of the known PDFs' MD5 drifted from baseline. Idempotency: each issue body carries a `<!-- signature: <sha1> -->` line; re-runs with the same findings find the existing open issue and skip filing — closing the issue clears the suppression, so if the signal persists next week's run reopens. (this commit)
- Quiz landing page shows a per-domain proficiency strip — 5 rows of `done / total · pct` with a prog-bar mirroring the sidebar widget. The 1-2 domains with the lowest done % (and pct < 100) get a `💡 weak` chip + orange accent so the user can see at a glance where they're behind. Paired with a new `🩺 Drill weak spots` quick-start preset that uncheck-all-domains-except-weak, force-enables `only-undone`, and fires a 10-question random draw — turning the proficiency dashboard into a one-click study loop. Reuses existing `quiz-start-btn` path so timer / status / summary flow are unchanged. If all domains are 100% done, the button alerts and aborts. (this commit)
- Browse sidebar gains a custom hover tooltip for section + exercise rows whose label is too long to fit the 280 px column. Replaces the native browser `title=` tooltip with a styled, fixed-positioned, multi-line, max 400 px wide bubble that appears instantly (no 1.5 s native delay) and follows the theme. Auto-flips to the left when the right edge would clip; auto-clamps vertically; hides on scroll / Esc / mouseleave. `aria-label` preserves screen-reader semantics. (this commit)

References [chadmcrowell/CKA-Exercises#cka-changes-2024](https://github.com/chadmcrowell/CKA-Exercises/blob/main/cka-changes-2024/README.md) as inspiration. (this commit)

### Fixed
- Release workflow's push to a Repository-Rule-protected `main` was failing with `GH013: Cannot update this protected ref`. Investigation chain: (1) the initial fix attempt added a new "bypass" ruleset with Repository admin — but the **existing** "Protect Main" ruleset (with the `update` rule) was the actual blocker, and (2) `github-actions[bot]` doesn't match any Role-based bypass entry — workflows aren't users, so "Repository admin" / "Maintain" Role bypasses don't apply to them. GitHub's Rulesets UI on this repo's plan also doesn't expose GitHub Actions as a bypass-able Integration option. Resolved by switching the release workflow to a fine-grained PAT stored as `secrets.RELEASE_PAT`: both `actions/checkout@v4` and the release step's `gh release create` now use it, so the push happens as the admin user identity which IS on the bypass list. CLAUDE.md `## Release workflow → One-time setup` rewritten with the correct PAT generation + secret-storage steps + 90-day rotation note. `scripts/release.mjs`'s GH013 hint (preserved) still points readers back to this section. (this commit)
- Desktop Chrome mode-switch lag (Quiz → Browse, Docs → Browse, etc.) was visible for ~50-150 ms because `renderBrowse()` always rebuilt the full sidebar tree (271 `<button>`s inside the 280 px sidebar column → real layout reflow on every render). Mobile didn't notice — the sidebar is hidden via CSS there, so detached-tree mutations skip the reflow. Now `renderBrowse()` keeps a `_sidebarFiltersSignature` over the structural inputs (domain checkboxes / tag checkboxes / search / only-bookmarks / only-undone / reveal-solutions); on signature match it runs the new `syncSidebarStateClasses()` (cheap class toggles + done-count text update per `tree-domain`, O(visible)) instead of the full `renderSidebar`. Quiz → Browse round-trip drops to ~1 ms. (this commit)
- `🩺 Drill weak spots` button silently did nothing on desktop Chrome (worked on mobile Safari). Root cause: the original quick-start implementation round-tripped through hidden form inputs (`cb.checked = …` then `quiz-start-btn.click()`), and that path was racy on desktop — the click on a button inside a `hidden=true` parent didn't always fire the submit handler. Replaced all three quick-starts (`🎲 10 random`, `🎯 17-question mock`, `🩺 Drill weak spots`) with a new `startQuizFromConfig(cfg)` entry point that builds the quiz state directly in memory from an explicit config object — no form input writes, no synthetic clicks. Form's `▶ Start quiz` button still works via the legacy `startQuiz()` reader, which now also delegates to `startQuizFromConfig`. (this commit)
- Header popovers (☁ Sync, 🐛 Issues, 🤖 LLM) are now mutually exclusive — opening one closes any other that's currently open. Two visible symptoms fixed: (a) opening LLM and then clicking Issues had no apparent effect (Issues opened but rendered underneath the still-visible LLM popover); (b) opening Issues and then clicking Sync left both popovers stacked. Root cause: each toggle's click handler called `e.stopPropagation()`, which prevented sibling popovers' document-level click-outside dismiss handlers from ever firing. Removing the three stopPropagation calls lets the existing dismiss handlers do their job — the popover's own dismiss handler early-returns when target is inside its own toggle/menu, so removing stopPropagation doesn't cause self-close. (this commit)
- Browse search bar input no longer freezes the page on common keywords. Root cause: every keystroke (including Backspace) ran the full renderBrowse pipeline — sidebar tree rebuild + main panel clear + `marked.parse()` per visible card with no caching — synchronously. Queries matching 100+ exercises took 500 ms–2 s of main-thread blocking, which is why the delete key didn't respond. Two fixes layered: (a) 200 ms trailing debounce on the search input so renderBrowse only runs after typing pauses; (b) per-source markdown cache (bounded LRU at 1000 entries) so the same task/solution markdown isn't re-parsed across keystrokes. Cache also benefits Quiz / Docs / Help / Tools / Settings render paths that hit `renderMarkdown` with static content. (this commit)
- `build-and-deploy-docs.yml` workflow's `paths` filter was missing `EXAM_GUIDE.md` / `EXAM_GUIDE_CN.md` / `CHANGELOG.md` — those three markdown files are bundled into `docs/exercises.json` at build time (Help mode reads them), but edits to them didn't trigger a Pages rebuild. Now added; doc-only edits to any of the bundled markdown files trigger a deploy. (this commit)
- Issue Queue mutations (🐞 Quick Flag, 💾 Save draft, 🗑 Remove) now trigger the 30 s auto-push debounce. Previously `setFixDraft` / `setTaskFixDraft` wrote to localStorage + stamped the keymeta side-table but didn't call `markSyncDirty()`, so flags only reached the gist when the user happened to also toggle Done / Bookmark / Answer / quiz state — solo flags never auto-synced. (this commit)
- Task-side 🐞 Quick Flag was silently no-op'ing. `setTaskFixDraft`'s empty-prune check didn't include `payload.flagged`, so `{flagged:true}` stubs got `removeItem`'d immediately after being written. Solution-side flags worked fine; task-side never persisted. (this commit)
- Task-side draft deletion (🗑 Remove on a `task-fix` queue item) didn't stamp a tombstone in keymeta. The merge engine then treated the missing local key as "never seen" and re-adopted the remote copy on next pull — same resurrection class as the earlier solution-side fix in 04ab81b. (this commit)
- Removed queue drafts no longer resurrect after a sync. `mergePayload`'s "other singletons" branch now respects local tombstone timestamps instead of greedily adopting remote whenever local lacked a value. ([04ab81b])
- iPhone "self-conflict" loop where the device's own `beaconPush` triggered a phantom remote-newer-than-baseline detection on next session. `bootAutoSync` now refreshes baseline from gist `updated_at` after a beacon push. ([3d482ed])
- Quiz fullscreen ☁ button click was opening and immediately closing the popover (click bubbled to the document-level dismiss handler). Now opens the popover directly via the closure exposed by `installSyncMenu`, bypassing the unreliable synthetic-click indirection. ([04ab81b])
- kubectl -h help text overflowing the right edge of the Tools drawer card on phones. ([da57e8f])
