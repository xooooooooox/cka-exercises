# Changelog

All notable changes to the CKA practice repository ‚Äî **webapp, exercise corpus, AND documentation** ‚Äî are recorded here. New entries land under `## [Unreleased]` until a tagged release.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/). Categories used:

- **Added** ‚Äî new features, new exercises, new docs sections
- **Changed** ‚Äî behavior or wording changes that aren't bug fixes
- **Fixed** ‚Äî bug fixes
- **Removed** ‚Äî deleted features, deprecated docs

Each entry references the commit hash in parens for traceability to git history. Root-cause notes / migration details live in commit messages; the changelog stays single-line and user-visible.

### Fixed
- Release workflow's push to a Repository-Rule-protected `main` was failing with `GH013: Cannot update this protected ref`. Investigation chain: (1) the initial fix attempt added a new "bypass" ruleset with Repository admin ‚Äî but the **existing** "Protect Main" ruleset (with the `update` rule) was the actual blocker, and (2) `github-actions[bot]` doesn't match any Role-based bypass entry ‚Äî workflows aren't users, so "Repository admin" / "Maintain" Role bypasses don't apply to them. GitHub's Rulesets UI on this repo's plan also doesn't expose GitHub Actions as a bypass-able Integration option. Resolved by switching the release workflow to a fine-grained PAT stored as `secrets.RELEASE_PAT`: both `actions/checkout@v4` and the release step's `gh release create` now use it, so the push happens as the admin user identity which IS on the bypass list. CLAUDE.md `## Release workflow ‚Üí One-time setup` rewritten with the correct PAT generation + secret-storage steps + 90-day rotation note. `scripts/release.mjs`'s GH013 hint (preserved) still points readers back to this section. (this commit)
- Desktop Chrome mode-switch lag (Quiz ‚Üí Browse, Docs ‚Üí Browse, etc.) was visible for ~50-150 ms because `renderBrowse()` always rebuilt the full sidebar tree (271 `<button>`s inside the 280 px sidebar column ‚Üí real layout reflow on every render). Mobile didn't notice ‚Äî the sidebar is hidden via CSS there, so detached-tree mutations skip the reflow. - `ü©∫ Drill weak spots` button silently did nothing on desktop Chrome (worked on mobile Safari).
